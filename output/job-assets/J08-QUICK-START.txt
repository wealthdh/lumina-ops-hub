================================================================================
WHAT IT DOES
================================================================================

Detects statistical arbitrage between Polymarket prediction markets and
traditional financial instruments (forex, crypto, equities).

Core algorithm:
  1. Fetch Polymarket markets from Gamma API
  2. Categorize markets (Fed cuts → EURUSD, BTC → BTCUSD, etc.)
  3. Compare Polymarket price vs financial market implied probability
  4. Flag opportunities with >2% probability spread
  5. Calculate Kelly Criterion position sizing
  6. Generate specific trade recommendations

Example opportunity:
  Market: "Will Fed cut rates in Q2 2026?"
  Polymarket price: $0.62 (62% probability)
  EURUSD implied: 58% probability
  Spread: 4% = 4.1% edge
  Trade: Buy YES at $0.62 + Short EURUSD = Hedged 4.1% gain
  Position: 2.05% of capital (Kelly fraction)

Keyword matching is automatic - no manual categorization needed.

================================================================================
POSITION SIZING (KELLY CRITERION)
================================================================================

Formula: f* = (b × p - q) / b
Where: p = win probability, q = loss probability (1-p), b = odds

Safety factor applied: Use 1/4 Kelly (reduces volatility)
Maximum position: 25% of capital per trade

Example:
  Spread: 7% (55% vs 48%)
  Kelly calculation: (1 × 0.55 - 0.45) / 1 = 10%
  Fractional Kelly (1/4): 10% × 0.25 = 2.5% of capital

Rationale: Smaller positions = less drawdown from edge estimation errors

================================================================================
TRADE EXECUTION
================================================================================

Each opportunity includes specific instructions:

Scenario A - Polymarket OVERPRICED:
  1. Buy YES on Polymarket at $X
  2. Short the financial instrument at current market price
  Result: Profits if outcome true (YES side) OR false (financial hedge)

Scenario B - Polymarket UNDERPRICED:
  1. Sell YES on Polymarket at $X
  2. Long the financial instrument at current market price
  Result: Profits if outcome false (NO side) OR true (financial hedge)

Risk: Minimal due to hedge (only spread captured as P&L)

================================================================================
FILTERING LOGIC
================================================================================

Threshold checks (auto-applied):
  - Probability spread >= 2.0% (minimum edge)
  - Only active/open markets (closed=False)
  - Price must be valid (0.0 - 1.0 range)

Risk classification:
  - Low risk: Edge < 5%
  - Medium risk: Edge >= 5%

Sorting: Opportunities sorted by edge % descending (best first)

================================================================================
ERROR HANDLING
================================================================================

API failures handled with:
  - 3 automatic retries (1-second backoff between attempts)
  - 10-second timeout per request
  - Graceful fallback (returns empty list if API unavailable)

Logging:
  - INFO: Cycle start, markets fetched, results
  - WARNING: Retry attempts, dropped markets
  - ERROR: API failures, processing errors

Console output continues even if API fails (shows "No opportunities detected")

================================================================================
CONFIGURATION TUNING
================================================================================

Adjust in j08-arb-detector.py:

  MIN_SPREAD_THRESHOLD = 0.02
    → Lower to 0.01 for tighter spreads
    → Higher to 0.03 for stronger signals only

  KELLY_SAFETY_FACTOR = 0.25
    → Lower (0.10) for more aggressive sizing
    → Higher (0.50) for more conservative sizing

  REQUEST_TIMEOUT = 10
    → Lower for faster fail detection
    → Higher for slower networks

  RETRY_MAX = 3
    → Increase for unreliable networks
